## 步骤一：上传项目文件

### 1.1 找到正确的部署包

在这个文件夹里，有两个版本：

* `smart-album-cloud/` —— 源代码版本（前端和后端分开，不能直接部署）
* `smart-album-deploy/` —— **部署版本（用这个！）**

### 1.2 上传文件

你需要把 `smart-album-deploy/` 文件夹里的**所有内容**上传到服务器的 `/www/wwwroot/47.108.232.40\_2026/cloud/` 目录下。

**具体操作（任选一种）：**

#### 方式A：用宝塔面板上传（推荐）

1. 登录宝塔面板
2. 点击左侧菜单「文件」
3. 进入 `/www/wwwroot/47.108.232.40\_2026`
4. 点击「上传」按钮，选择你电脑里的 `smart-album-deploy` 文件夹压缩包（先压缩成 zip）
5. 上传后右键 zip 文件 → 「解压」
6. **重命名解压后的文件夹为 `cloud`**（如果不是这个名字的话）

#### 方式B：用 FTP / SFTP 上传

用 WinSCP、FileZilla 等工具连接服务器，把 `smart-album-deploy/` 里的所有文件上传到 `/www/wwwroot/47.108.232.40\_2026/cloud/`

### 1.3 上传后的目录结构应该是这样

```
/www/wwwroot/47.108.232.40\_2026/cloud/
├── index.html              # 登录页
├── register.html           # 注册页
├── dashboard.html          # 控制台
├── albums.html             # 相册管理
├── devices.html            # 设备管理
├── css/
│   └── style.css           # 样式文件
├── backend/
│   ├── api/                # API接口
│   │   ├── auth/           # 登录注册相关
│   │   ├── albums/         # 相册相关
│   │   ├── devices/        # 设备相关
│   │   ├── upload/         # 上传相关
│   │   └── user/           # 用户信息
│   ├── config/
│   │   └── database.php    # 数据库配置文件 ⭐重要
│   └── includes/
│       └── functions.php   # 公共函数
└── uploads/                # 图片上传目录（后面会创建）
```

## 步骤二：修改后端配置文件

这是最核心的一步！你需要告诉网站：数据库密码是多少、MQTT地址是什么、网站地址是什么。

### 2.1 打开配置文件

在宝塔面板中：

1. 进入文件管理器
2. 找到 `/www/wwwroot/47.108.232.40\_2026/cloud/backend/config/database.php`
3. 双击打开编辑

### 2.2 需要修改的地方

找到下面这几行，按你的实际情况填写：

```php
// 数据库配置（必填）
define('DB\_HOST', 'localhost');      // 数据库地址，一般保持 localhost
define('DB\_NAME', 'smart\_album');    // 数据库名，可以改，但要和后面导入的一致
define('DB\_USER', 'root');           // 数据库用户名
define('DB\_PASS', '你的数据库密码');  // ⚠️ 改成你数据库的密码！
define('DB\_CHARSET', 'utf8mb4');

// MQTT配置（EMQX已装好，必填）
define('MQTT\_HOST', 'localhost');    // EMQX的IP地址，如果在同一台服务器就写 localhost
define('MQTT\_PORT', 1883);           // MQTT端口，默认1883
define('MQTT\_USER', '');             // 如果EMQX设置了认证，就填用户名
define('MQTT\_PASS', '');             // 如果EMQX设置了认证，就填密码

// 应用配置（必填）
define('APP\_URL', 'http://47.108.232.40:80');  // ⚠️ 改成你的实际访问地址！
```

#### `APP\_URL` 怎么填？

* 如果你用 IP+端口 访问，比如 `http://47.108.232.40:8080`，就填这个
* 如果你用域名访问，比如 `http://your-domain.com`，就填域名
* **注意**：后面设备下载图片会用到这个地址，所以一定要能外网访问！

#### `UPLOAD\_PATH` 和 `UPLOAD\_URL` 不要改！

这两行已经配置好了，保持原样：

```php
define('UPLOAD\_PATH', \_\_DIR\_\_ . '/../../uploads/');
define('UPLOAD\_URL', '/uploads/');
```

> 如果你是用 IP 的某个端口直接访问 `http://IP:端口/cloud/`，那 `UPLOAD\_URL` 可能需要改成 `/cloud/uploads/`，这个后面测试时再说。

### 2.3 保存文件

修改完后，点击宝塔编辑器的「保存」按钮。

## 步骤三：创建数据库并导入数据

### 3.1 创建数据库

**方式A：用宝塔面板创建**

1. 宝塔面板 → 「数据库」
2. 点击「添加数据库」
3. 数据库名填：`smart\_album`
4. 用户名和密码自己设置，**记住密码**
5. 字符集选 `utf8mb4`
6. 点击「提交」

**方式B：用 phpMyAdmin 创建**

1. 宝塔面板 → 「数据库」→ 点击你服务器的 phpMyAdmin 入口
2. 登录后，点击「数据库」→「新建」
3. 数据库名填 `smart\_album`
4. 字符集选 `utf8mb4\_unicode\_ci`
5. 点击「创建」

### 3.2 导入 SQL 文件

1. 在 phpMyAdmin 中，点击左边刚创建的 `smart\_album` 数据库
2. 点击顶部菜单「导入」
3. 点击「选择文件」，找到项目里的 `smart-album-cloud/database/smart\_album.sql`
4. 下滑点击「执行」

### 3.3 导入成功后会有的内容

* `users` 用户表（含默认管理员账号）
* `albums` 相册表
* `images` 图片表
* `devices` 设备表
* `device\_image\_logs` 下发记录表

**默认管理员账号：**

* 用户名：`admin`
* 密码：`admin123`

> ⚠️ 上线后请立即修改默认密码！

## 步骤四：创建 uploads 目录并设置权限

图片上传后都存到这个目录，必须让 PHP 有写入权限。

### 4.1 创建目录

在宝塔文件管理器中：

1. 进入 `/www/wwwroot/47.108.232.40\_2026/cloud/`
2. 点击「新建文件夹」
3. 文件夹名填 `uploads`

### 4.2 设置权限

1. 右键点击 `uploads` 文件夹
2. 选择「权限」
3. 设置为 `755` 或 `777`（测试阶段建议 `777`，上线后改 `755`）
4. 勾选「应用到子目录」（如果有这个选项）
5. 点击「确定」

> Linux 命令行方式：
> ```bash
> chmod -R 777 /www/wwwroot/47.108.232.40\_2026/cloud/uploads
> ```

## 步骤五：Nginx 配置（重点）

### 5.1 宝塔里的基本设置

因为你已经在宝塔建了一个网站，根目录是 `/www/wwwroot/47.108.232.40\_2026`，那么：

1. 进入宝塔 → 「网站」
2. 找到你建好的这个站点，点击「设置」
3. 确认「网站目录」是 `/www/wwwroot/47.108.232.40\_2026`
4. 运行目录保持默认即可（不要改成 `/cloud`，除非你用子目录绑定）

### 5.2 配置 URL 重写 / 伪静态

由于前端页面都在 `/cloud/` 目录下，你需要让用户能直接访问到。

**如果你是用 IP 的某个端口，直接访问 `http://IP:端口/cloud/`：**

这种情况一般不需要额外配置，因为 `/cloud/` 本来就是一个真实存在的目录。

**如果你想把 `http://IP:端口/` 直接指向 `/cloud/`：**

那需要修改运行目录。在宝塔站点设置里：

* 点击「网站目录」
* 把「运行目录」改成 `/cloud`
* 保存

这样访问 `http://IP:端口/` 就直接打开登录页了。

### 5.3 配置 PHP 支持

确保你的站点使用的是 PHP 8.1：

1. 宝塔站点设置 → 「PHP版本」
2. 选择 `PHP-8.1`
3. 保存

### 5.4 配置 uploads 目录可访问

Nginx 需要让上传的图片能被浏览器直接访问。

在宝塔站点设置里：

1. 点击「配置文件」（Nginx 配置）
2. 找到 `server { }` 块
3. 在里面添加一段：

```nginx
    location /cloud/uploads/ {
        alias /www/wwwroot/47.108.232.40\_2026/cloud/uploads/;
        try\_files $uri $uri/ =404;
    }
```

> 如果你的运行目录已经改成 `/cloud`，那就写：
> ```nginx
>     location /uploads/ {
>         alias /www/wwwroot/47.108.232.40\_2026/cloud/uploads/;
>         try\_files $uri $uri/ =404;
>     }
> ```

4. 点击保存
5. 回到网站列表，点击「重载配置」或「重启」Nginx

## 步骤六：安装 PHP GD 扩展（验证码需要）

验证码图片需要 PHP 的 GD 库支持。

### 宝塔安装方式：

1. 宝塔面板 → 「软件商店」
2. 找到已安装的 `PHP-8.1`
3. 点击「设置」
4. 选择「安装扩展」
5. 找到 `gd`，点击「安装」
6. 安装完成后，重载 PHP 配置（或重启 PHP 服务）

### 命令行验证方式：

```bash
php -m | grep gd
```

如果有输出 `gd` 就是安装成功了。

## 🔧 常见问题排查

### 1\. 验证码不显示

* 检查 PHP 8.1 的 `gd` 扩展是否安装
* 检查 `backend/api/auth/captcha.php` 是否能直接访问

### 2\. 登录时提示 "网络错误"

* 检查 `index.html` 里的 API 地址是不是 `backend/api/...`（不能是 `../backend/api/...`）
* 检查浏览器 F12 → Network 看看具体请求报什么错

### 3\. 数据库连接失败

* 检查 `backend/config/database.php` 里的密码是否填对
* 检查数据库名 `smart\_album` 是否已创建
* 检查 MySQL 是否正常运行

### 4\. 图片上传成功但不显示

* 检查 `uploads/` 目录权限是否为 `755` 或 `777`
* 检查 Nginx 配置中 `uploads` 的 `alias` 路径是否正确
* 检查 `database.php` 中的 `APP\_URL` 和 `UPLOAD\_URL` 是否正确

### 5\. MQTT 下发图片失败

* 检查 EMQX 是否正常运行：`systemctl status emqx`
* 检查 `database.php` 中 `MQTT\_HOST` 和 `MQTT\_PORT` 是否正确
* 检查服务器防火墙是否放行了 1883 端口

1. **修改默认管理员密码** 或删除默认 `admin` 账号
