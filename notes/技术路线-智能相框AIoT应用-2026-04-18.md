# 智能相框AIoT应用技术路线文档

**生成时间**: 2026-04-18 20:28

## 1\. 项目概述

### 1.1 项目名称

智能相框云平台 (SmartFrameCloud)

### 1.2 项目定位

基于Web的智能相框AIoT应用，提供图片云端存储、管理和向智能相框设备分发的完整解决方案。

### 1.3 核心功能

* 用户认证管理（登录/注册）
* 相册管理（创建、删除、查看）
* 图片上传与管理
* 智能相框设备绑定与管理
* 图片向设备的MQTT分发
* 图片处理（转换为BMP格式）

## 3\. 核心模块设计

### 3.1 用户认证模块

```
认证流程：
1. 用户输入账号密码 + 验证码
2. 后端验证验证码（5分钟有效期）
3. 验证用户凭据（支持用户名/邮箱登录）
4. 设置Session
5. 返回用户信息

API端点：
- POST /backend/api/auth/login.php
- POST /backend/api/auth/register.php
- POST /backend/api/auth/logout.php
- GET /backend/api/auth/captcha.php
```

### 3.2 相册管理模块

```
功能特性：
- 创建相册（名称、描述）
- 删除相册级联图片
- 查看相册列表
- 相册封面显示
- 图片统计

数据表：
- albums: 相册信息
- images: 图片信息（关联相册）

API端点：
- POST /backend/api/albums/create.php
- POST /backend/api/albums/delete.php
- GET /backend/api/albums/list.php
- GET /backend/api/albums/images.php
```

### 3.3 图片处理模块

```
处理流程：
1. 图片上传验证（格式、大小）
2. 等比缩放+居中裁剪（800x480）
3. 7色有序抖色处理
4. 转换为24位BMP格式
5. 缓存机制

技术特点：
- GD库图像处理
- 自定义调色板（7色）
- Bayer矩阵抖动算法
- 缓存文件管理

API端点：
- POST /backend/api/upload/upload.php
- GET /backend/api/imgprocess/imgprocess.php
```

### 3.4 设备管理模块

```
功能特性：
- 设备绑定（64位芯片ID）
- 设备状态管理（在线/离线）
- 图片下发到设备
- 设备解绑
- MQTT通信

MQTT通信协议：
- Topic: device/{device\_uid}/image
- Payload: JSON格式图片信息
- QoS: 0（最多一次）

API端点：
- POST /backend/api/devices/bind.php
- POST /backend/api/devices/unbind.php
- GET /backend/api/devices/list.php
- POST /backend/api/devices/send-image.php
```

## 4\. 数据库设计

### 4.1 核心数据表

#### users表

```sql
CREATE TABLE users (
    id INT PRIMARY KEY AUTO\_INCREMENT,
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    password\_hash VARCHAR(255) NOT NULL,
    status TINYINT DEFAULT 1 COMMENT '1:启用 0:禁用',
    created\_at DATETIME DEFAULT CURRENT\_TIMESTAMP,
    last\_login\_at DATETIME NULL
);
```

#### albums表

```sql
CREATE TABLE albums (
    id INT PRIMARY KEY AUTO\_INCREMENT,
    user\_id INT NOT NULL,
    name VARCHAR(100) NOT NULL,
    description TEXT,
    created\_at DATETIME DEFAULT CURRENT\_TIMESTAMP,
    updated\_at DATETIME DEFAULT CURRENT\_TIMESTAMP ON UPDATE CURRENT\_TIMESTAMP,
    FOREIGN KEY (user\_id) REFERENCES users(id)
);
```

#### images表

```sql
CREATE TABLE images (
    id INT PRIMARY KEY AUTO\_INCREMENT,
    album\_id INT NOT NULL,
    user\_id INT NOT NULL,
    original\_name VARCHAR(255) NOT NULL,
    file\_name VARCHAR(255) NOT NULL,
    file\_path VARCHAR(255) NOT NULL,
    file\_size INT NOT NULL,
    mime\_type VARCHAR(50) NOT NULL,
    width INT,
    height INT,
    created\_at DATETIME DEFAULT CURRENT\_TIMESTAMP,
    FOREIGN KEY (album\_id) REFERENCES albums(id),
    FOREIGN KEY (user\_id) REFERENCES users(id)
);
```

#### devices表

```sql
CREATE TABLE devices (
    id INT PRIMARY KEY AUTO\_INCREMENT,
    user\_id INT NOT NULL,
    device\_uid VARCHAR(16) UNIQUE NOT NULL COMMENT '64位芯片ID(16位十六进制)',
    name VARCHAR(100),
    description TEXT,
    status TINYINT DEFAULT 1 COMMENT '1:在线 2:离线 0:未绑定',
    current\_image\_id INT NULL,
    bound\_at DATETIME NULL,
    created\_at DATETIME DEFAULT CURRENT\_TIMESTAMP,
    FOREIGN KEY (user\_id) REFERENCES users(id),
    FOREIGN KEY (current\_image\_id) REFERENCES images(id)
);
```

#### device\_image\_logs表

```sql
CREATE TABLE device\_image\_logs (
    id INT PRIMARY KEY AUTO\_INCREMENT,
    device\_id INT NOT NULL,
    image\_id INT NOT NULL,
    user\_id INT NOT NULL,
    status TINYINT DEFAULT 0 COMMENT '0:待发送 1:成功 2:失败',
    error\_message TEXT,
    created\_at DATETIME DEFAULT CURRENT\_TIMESTAMP,
    FOREIGN KEY (device\_id) REFERENCES devices(id),
    FOREIGN KEY (image\_id) REFERENCES images(id),
    FOREIGN KEY (user\_id) REFERENCES users(id)
);
```
